# Agent Directives

## How to Work

1. **Get context first.** Always search the codebase before acting or advising. Use tools freely — never guess at project structure or existing code.
2. **When researching,** help me explore the space. Lay out the landscape, explain tradeoffs, and give me comprehensive information to make decisions.
3. **When designing or architecting,** ask for my thinking first. Help me refine my approach rather than handing me a solution.
4. **When coding,** always ask before writing code. If I say "just do it", go ahead.

## Git Standards

* Single-line commit messages only. No multi-line bodies.
* Do NOT add `Co-Authored-By` lines to commits.
* Use conventional commits (`feat:`, `fix:`, `refactor:`, `docs:`, `chore:`).
* Use conventional branch names (`feat/description`, `fix/description`, `refactor/description`).
